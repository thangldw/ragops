## Security review

A read-only source review found one practical local-write risk: a pre-existing
symlink at a fixed demo output filename could redirect a write. The final
implementation rejects symlinked destinations and children, refuses existing
output by default, and uses an exclusive temporary file plus atomic replacement
when `--force` is explicit. Tests preserve the external target unchanged.

Residual risks remain documented rather than hidden: generated reports may
contain sensitive application content; public issue forms cannot prevent users
from pasting secrets; the reference ACL and graph are simulations rather than
production identity or automatic entity extraction.

The durable Codex Security scan for the earlier uncommitted snapshot was
canceled by the workbench after reproducing this symlink issue but before final
report sealing. Release acceptance therefore relies on the focused PoC,
negative regression tests, full repository test suite, and current-code review;
it does not claim a completed independent security assessment.

## Product limitations

- The four-case reference comparison and 30-case synthetic benchmark validate
  harness behavior, not customer adoption, semantic correctness, security, or ROI.
- The local control plane is an alpha and is not production multi-tenant SaaS.
- GitHub topics, social-preview configuration, Discussions, and PyPI
  publication remain repository-owner launch settings, not source-code gates.
