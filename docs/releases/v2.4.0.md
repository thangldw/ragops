# RAGOps v2.4.0 — adoption experience refresh

RAGOps v2.4.0 makes the product easier to understand and try without changing
evaluation semantics.

## Highlights

- A new developer-proof showcase centered on install, PASS baseline, BLOCK
  candidate, and portable JSON/Markdown/HTML evidence.
- Copyable five-minute demo commands and responsive release-report UI.
- A shorter decision-first README and rewritten onboarding, demo, portfolio,
  presentation, video, CV/LinkedIn, and launch materials.
- New implementation-derived adoption screenshot and design-QA evidence.

## Compatibility

Presentation-only additive release. Public schemas, metric meaning, gate
thresholds, CLI/API behavior, and open-core boundaries are unchanged.

## Distribution status

The GitHub Release, refreshed GitHub Pages showcase, and PyPI v2.4.0 are public.
The PyPI wheel and source archive match the GitHub Release SHA-256 values
byte-for-byte; a clean PyPI install reports 2.4.0 and completes `ragops demo`.

## Known limits

The new material improves discoverability but is not evidence of increased
adoption. Real conversion, repeat use, and willingness to pay require observed
design-partner data. Security testing remains deferred by owner instruction.
