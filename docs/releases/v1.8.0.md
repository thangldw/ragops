# RAGOps v1.8.0 — review visibility and measured adoption

RAGOps v1.8.0 makes release evidence easier to review in pull requests and adds
a portable, offline contract for measuring design-partner outcomes. Evaluation
semantics and the dependency-free core remain unchanged.

## Highlights

- Publish one bounded, idempotently updated pull-request comment from an
  isolated `workflow_run` workflow without executing untrusted PR artifacts.
- Verify repository, workflow, run, commit, pull request, artifact identity,
  file allowlist, sizes, UTF-8, and evidence manifest before publication.
- Record pseudonymous baseline and assisted pilot observations without raw
  prompts, customer documents, credentials, or personal identifiers.
- Generate deterministic JSON or Markdown evidence for activation, repeat
  usage, task-success uplift, time saved, incidents, reviewer disagreement,
  measured costs, and optional estimated ROI.
- Keep synthetic evidence visibly separated from real partner evidence and
  economic assumptions.

## Compatibility

This is an additive minor release. Existing evaluator, scenario, trace,
response, report, API, regression-policy, metric, and PASS/BLOCK contracts are
unchanged. The open-source core remains useful offline with zero runtime
dependencies and Python 3.11 or newer.

## Known limits

The included pilot evidence is synthetic and does not demonstrate customer
adoption, causal impact, willingness to pay, or actual ROI. Live GitHub evidence
proves both PASS and BLOCK comment creation plus idempotent update; it does not
replace an adopter's own workflow, permissions, retention, and fork-PR review.
