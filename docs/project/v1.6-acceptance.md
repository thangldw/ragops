# v1.6 pull-request adoption acceptance

## Decision

**GO on `main`. The reusable workflow passed its GitHub-hosted smoke run. HOLD
a v1.6 release until the product owner chooses the next release boundary, and
hold PyPI publication until external ownership is configured.**

## Contract impact

The slice is additive. It introduces GitHub workflow adapters, print CSS, and
integration documentation. Scenario, trace, report, evaluator, metric, CLI,
API, and release-decision semantics do not change.

## Limitations

- The reusable workflow is present on `main` after v1.5.0 and passed run
  `29194652319`; production callers should pin a reviewed commit or a future
  release containing it.
- The workflow does not post PR comments and intentionally avoids write
  permissions or `pull_request_target` execution.
- PyPI still requires project ownership, the `pypi` GitHub environment, and a
  matching Trusted Publisher before the manual workflow can succeed.
- Design-partner feedback and adoption/ROI remain external evidence.
