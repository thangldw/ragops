# v1.7 broader-adoption acceptance

## Decision

**GO for deployment to `main`. GitHub discovery actions are verified. HOLD a
v1.7 release until the product owner selects the next release boundary.**

## Contract impact

The slice is additive: one demo selection, one evaluator class, and two CLI
options. Existing default demo, scenario, trace, response, report, evaluator,
API, regression-policy, metric, and release-decision semantics remain
compatible.

## Acceptance evidence

| Gate | Result | Evidence |
| --- | --- | --- |
| Default demo compatibility | Pass | Omitted scenario still writes Japanese troubleshooting bundle |
| Support-triage demo | Pass | Baseline passes; unsafe candidate is blocked with named gates |
| Unicode boundary | Pass | Japanese answer equal to limit passes at four code points |
| Negative configuration | Pass | Zero, negative, and boolean limits are rejected |
| Diagnostic behavior | Pass | Over-budget finding is medium and does not silently block release |
| CLI integration | Pass | Configured limit appears in JSON evaluation evidence |

## Limitations

- Both demos are synthetic and do not prove customer adoption or ROI.
- Code-point count is not token count, display width, or semantic quality.
- Enforcing length as a release gate requires a future reviewed policy
  contract; this release keeps it diagnostic.
