# v2.0 trustworthy extensible release-gates acceptance

## Decision

**GO for release 2.0.0. The owner authorized staged implementation and release
on 2026-07-13. Public publication remains conditional on release validation and
green GitHub CI.**

## Contract impact

Custom metric and severity gates are additive and opt-in. Existing metric
meanings and omitted-policy decisions remain unchanged. Runtime loaders now
enforce the numeric ranges already published in schemas. Protected API
endpoints change from implicit fail-open to explicit fail-closed authentication;
this incompatible correction establishes the 2.0 major boundary.

## Limitations

- No security scan or penetration test was run, by explicit owner instruction.
- Lexical claim support is not semantic entailment.
- API limits are process-level safety bounds, not distributed rate limiting or
  tenant quota enforcement.
- The synthetic benchmark does not establish Japanese model quality, customer
  adoption, or ROI.
- Hosted SSO/RBAC, tenant isolation, immutable audit, HA, and operational
  controls remain future work.
