# RAGOps v1.5.0 — adoption-first release evidence

RAGOps v1.5.0 makes the open-source release-gate workflow easier to understand,
run, review, and extend without adding runtime dependencies to the core.

## Highlights

- Run `ragops demo --output ragops-demo` without credentials or network access.
- Inspect portable scenario, baseline, candidate, Markdown, and HTML evidence.
- Compare the four-case Graph+ACL reference deployment with a lexical-only
  regression and the separate 30-case Japanese synthetic benchmark.
- Use the redesigned product showcase, README infographics, product screens,
  contributor guidance, support policy, and structured issue forms.
- Reuse the project under the MIT License; previously published Apache-2.0
  releases retain their original license.
- Consume a valid trace 0.4 JSON Schema; its documented fields and semantics are
  unchanged.

## Security

Demo output now fails closed for an existing destination. Explicit `--force`
replacement rejects symlinked destinations and fixed-name children, uses
exclusive temporary-file creation, and atomically replaces regular files.

## Compatibility

This is an additive minor release. Scenario, trace, report, metric, evaluator,
and release-gate semantics remain compatible. The dependency-free core still
requires Python 3.11 or newer.

## Known limits

The bundled evidence is synthetic and validates harness behavior; it does not
establish semantic correctness, production security, customer adoption, or ROI.
The local control plane remains an alpha rather than production multi-tenant
SaaS. PyPI publication is not part of this GitHub release.
