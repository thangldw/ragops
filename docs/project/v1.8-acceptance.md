# v1.8 review visibility and measured-adoption acceptance

## Decision

**GO for owner review and deployment to `main`. HOLD a versioned release until
the owner explicitly selects the next release boundary.**

## Contract impact

The change is additive. The GitHub evidence artifact gains a bounded metadata
manifest; existing report and log files remain. Pilot manifest, observation,
economics, report, and CLI contracts are new at version 0.1. Existing evaluator,
scenario, trace, response, report, API, policy, and PASS/BLOCK semantics do not
change.

## Acceptance evidence

| Gate | Result | Evidence |
| --- | --- | --- |
| Publisher trust boundary | Pass | Default-branch `workflow_run`; no PR checkout, artifact execution, cache, or `pull_request_target` |
| Permissions | Pass | Only `actions: read`, `contents: read`, and `pull-requests: write` |
| Supply chain | Pass | Checkout, setup-python, and upload-artifact uses are pinned to full commits |
| Artifact validation | Pass | Repository/workflow/run/SHA/PR/conclusion, exact files, sizes, UTF-8, JSON, redirect host, and manifest verified |
| Idempotency | Pass | One bot marker is created or updated; duplicate markers fail closed |
| Pilot contract | Pass | Strict manifest, JSONL observations, optional economics, pseudonymous cohort, and consent state |
| KPI decision | Pass | Repeat usage, success uplift, time saved, and zero-incident guardrail produce deterministic SCALE/HOLD |
| Evidence discipline | Pass | Measured observations and economic estimates are separated; synthetic report is labeled |
| CLI path | Pass | Synthetic pilot generates synchronized Markdown evidence end to end |
| Core quality | Pass | Ruff clean and 99 automated tests pass |

## Limitations

- Local tests validate the publisher contract; live PR-comment publication must
  be verified after the workflow exists on the default branch.
- The included pilot is synthetic and does not demonstrate a real design
  partner, causal impact, customer adoption, willingness to pay, or actual ROI.
- A real pilot requires partner consent, a frozen cohort/period/target contract,
  reviewed pseudonymous observations, and owner approval before public claims.
