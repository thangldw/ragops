# Competitive positioning

RAGOps competes with a workflow assembled from evaluation libraries, tracing
platforms, security scanners, custom notebooks, and CI scripts. The wedge is
not “more metrics”; it is a portable and explainable release contract.

| Alternative | Strength | Gap RAGOps targets |
| --- | --- | --- |
| Evaluation libraries | Rich metric catalog | Release policy and regression workflow are often application code |
| Hosted observability | Strong trace exploration | Private/offline use and portable contracts may be secondary |
| Prompt testing tools | Good developer iteration | Enterprise evidence and critical policy gates may be limited |
| Security scanners | Deep adversarial testing | Quality, operational budget, and baseline comparison are separate |
| Custom notebooks | Flexible | Difficult to version, review, reproduce, and gate in CI |

## Differentiation thesis

1. Local-first and provider-neutral core.
2. Scenario, trace, and report contracts are portable and versioned.
3. Absolute gates and baseline regression gates are equally important.
4. Deterministic evidence is always visible alongside model-based judgment.
5. FDE-style scenario design connects customer outcomes to engineering checks.

This document is a product hypothesis, not a claim that competitors lack every
listed capability. A sourced market review is required before external use.
