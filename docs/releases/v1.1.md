# Catch RAG and agent regressions before they reach production

![RAGOps pull-request comment showing a blocked candidate](https://raw.githubusercontent.com/thangldw/ragops/v1.1/docs/demo/ragops-pr-comment.png)

RAGOps 1.1 makes the first useful result visible in one command and carries the
same evidence into pull requests.

- Run `uvx ragops demo` without cloning, an account, or a model credential.
- Add the reusable gate in seven YAML lines.
- See baseline/candidate metric deltas, named block reasons, and a standalone
  HTML report in the PR evidence artifact.
- Reproduce six named Failure Zoo specimens, including stale-source and
  failure-to-abstain contracts with explicit lexical boundaries.
- Use the repository's own BLOCK→PASS acceptance evidence from
  [PR #26](https://github.com/thangldw/ragops/pull/26) and the immutable
  [blocked workflow run](https://github.com/thangldw/ragops/actions/runs/29495688732).

The fixtures are synthetic. They validate the harness and release workflow, not
semantic correctness, production security, customer adoption, or ROI.
