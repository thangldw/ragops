# RAGOps v1.6.0 — pull-request adoption path

RAGOps v1.6.0 makes it easier to turn application traces into reviewable AI
release evidence and enforce the same decision in pull requests.

## Highlights

- Call a reusable, read-only GitHub Actions release gate with scenario,
  baseline, candidate, and optional policy inputs.
- Preserve PASS/BLOCK exit behavior while always publishing a Step Summary,
  command log, and evidence artifact.
- Print standalone HTML comparison reports with legible A4 rules and complete
  decision, metric, gate, and case details.
- Export a first portable trace through a credential-free tutorial.
- Convert synthetic OpenTelemetry spans into trace 0.4 JSONL through a strict,
  dependency-free example adapter.
- Prepare future PyPI publication through a manual, commit-pinned OIDC Trusted
  Publishing workflow; registry ownership is intentionally still required.

## Compatibility

This is an additive minor release. Scenario, trace, report, evaluator, metric,
CLI, API, and release-decision semantics remain compatible. The open-source
core still has zero runtime dependencies and requires Python 3.11 or newer.

## Known limits

The reusable workflow intentionally does not post write-enabled pull-request
comments. The OpenTelemetry adapter consumes pre-filtered exported spans; it is
not an SDK or collector. The bundled evidence is synthetic and does not prove
production security, customer adoption, or ROI. PyPI publication is not part
of this GitHub release.
