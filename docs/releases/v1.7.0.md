# RAGOps v1.7.0 — broader adoption proof

RAGOps v1.7.0 broadens the credential-free product proof beyond Japanese
troubleshooting while preserving the dependency-free evaluation core and
existing release semantics.

## Highlights

- Run a synthetic support-triage demo whose unsafe candidate is blocked for an
  unsupported refund action and missing evidence.
- Run a synthetic proposal-review demo whose candidate is blocked for inventing
  a data-residency requirement and omitting its citation.
- Add a deterministic `answer_length_budget` evaluator with Unicode code-point
  counting, CLI configuration, diagnostics, and negative validation.
- Copy a GitLab merge-request gate that preserves the RAGOps exit code and
  uploads its command log and evidence even when the gate blocks a release.
- Review a least-privilege design for future GitHub pull-request comments while
  keeping the shipped workflow read-only.
- Discover the project through improved GitHub metadata, Discussions, and three
  completed starter issues.

## Compatibility

This is an additive minor release. Existing default demo behavior, scenario,
trace, response, report, evaluator, CLI, API, metric, regression-policy, and
release-decision contracts remain compatible. The open-source core still has
zero runtime dependencies and requires Python 3.11 or newer.

## Known limits

All bundled workflows and benchmark evidence are synthetic; they do not prove
customer adoption, production security, or ROI. Answer length counts Unicode
code points rather than tokens, display width, or semantic quality and remains
diagnostic unless a reviewed policy explicitly makes it a release gate. The
GitHub PR-comment publisher is a design only: v1.7.0 does not add write-enabled
workflow permissions.
