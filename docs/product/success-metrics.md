# Success metrics

## North-star metric

**Defensible decisions per active project per month:** release or hold decisions
backed by a versioned scenario and persisted report.

## Activation

- Time from installation to first evaluation: target under 10 minutes.
- New projects completing one baseline/candidate comparison: target 60%.
- Sample scenario passing without external credentials: target 100%.

## Engagement and retention

- Weekly active projects running at least one evaluation.
- Scenarios with more than one accepted baseline revision.
- Percentage of failed production cases converted into regression fixtures.

## Product quality

- Identical inputs produce identical deterministic reports.
- Contract-breaking changes require a schema version change.
- Core test and lint pass rate remains 100% on supported Python versions.
- Median local deterministic run remains below one second for 1,000 cases.

## Commercial signals

- Teams requesting shared datasets, SSO/RBAC, audit retention, or hosted runs.
- Number of projects with more than five contributors.
- Conversion from open-source project to hosted workspace evaluation.

These are proposed targets. Public claims must use measured repository or
product telemetry, never illustrative values presented as real adoption.
